Learning Perl, Sixth Edition: Chapters 1-3

Written by Tracy Poff in misc on Fri 16 October 2015. Tags: perl, Learning Perl,

I needed to brush up on my perl (haven’t used it for anything in a decade—brush up might not be precisely the right term), and my minor misadventures with it have convinced me that I should do the thing properly, so I’m going to go through Learning Perl, Sixth Edition (‘the Llama book’), perhaps followed by some other recommended texts. Reading yet another programming text book doesn’t sound too exciting, but (I tell myself) strong foundations are important.

Here follow, therefore, my notes on the book (Chapters 1-3).

Chapter 1: Introduction

This is more or less a standard introductory chapter, though with rather more personality than usual. The exercises ask the reader first to type in a standard ‘hello world’ program, then a marginally more interesting program that reformats the output of perldoc.

I give this chapter points for style, but we’ve all read these kinds of things a hundred times, so it’s a relief to be on to…

Chapter 2: Scalar Data

This chapter looks at the ways of specifying numbers and strings, and some operations on them. Also: use warnings;, booleans (and if), user input (and chomp), while, undef (and defined).

Numbers

Numbers are all floating point in perl.

Strings

Single-quoted strings mean exactly what they say, except that \\ means a single backslash, and \' means a single quote. In particular, interpolation doesn’t happen and \n doesn’t mean a newline—it just means those two characters.

Double-quoted strings let you use backslash to make special characters (or any character) and they interpolate variables, so "foo $bar baz" gets the value of $bar substituted in. You can do something like "foo ${ba}r baz" if you need to (in this case) use the value $ba followed by the character r.

  • Concatenate strings with . infix: 'foo' . 'bar' == 'foobar'
  • Multiply strings with x infix: 'fred' x 3 == 'fredfredfred'

Chapter 3: Lists and Arrays

  • The range operator ..: (3..6, 12) == (3, 4, 5, 6, 12)
  • push and pop values at the end of a list
    • push(@array, $new_val)
    • push(@array, @several_new_vals)
    • pop @array
  • unshift and shift values at the front of a list
  • splice (@array, $start_index, $length_to_remove, @replacement_array) returns the removed items
  • reverse arrays with @array = reverse @array
  • @sorted = sort @array
  • while( my( $index, $value ) = each @rocks ) { say "$index: $value"; } with use 5.012;
